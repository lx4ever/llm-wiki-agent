## 提取内容

别把OpenClaw当ChatGPT用了，手把手教你

- 作者强调 OpenClaw 的价值在于多 agent、多场景协作，而不是单一聊天窗口

- 不同 agent 的隔离不一定靠不同 channel，也可以靠不同群

- 同一个机器人可以服务多个 agent，关键是把它拉进不同群并指定 agent

- 飞书群里是否必须 @bot，可以通过配置调整

- 评论区确认 Claude API 可以作为 OpenClaw 的主力模型使用

【这条在讲什么】
这是一条 OpenClaw 多 Agent 使用教程/思路分享。作者核心观点是，如果还只是把 OpenClaw 当成单一聊天机器人来用，很难发挥它的真正价值。重点不是一个 bot 聊天，而是创建 agent、做角色隔离、按群或场景分工。

【正文核心】
作者写了一个 OpenClaw 创建 Agent 的教程，明确强调单 Agent 用法很难发挥价值，多 agent、分场景使用才更接近 OpenClaw 的强项。

【评论区提炼出的关键信息】
- 隔离方式：不一定要用不同 channel 区分，用不同的群就可以，作者自己主要在 Telegram 里用不同群做区分。
- 机器人绑定：不需要每个 agent 一个不同 bot。可以是同一个机器人，把它拉进不同群，并在不同群里指定 agent。
- 群聊使用：建群本身不依赖机器人，但如果这个群要用 OpenClaw，就需要把机器人拉进去。
- 飞书问题：如果群里必须 @bot 才能收到消息，可以改 OpenClaw 配置，设成不需要 @。
- Claude API：评论区确认 Claude API 可以直接作为 OpenClaw 的主力模型使用。

【保守总结】
这条内容真正的重点不是装一个 OpenClaw，而是把 OpenClaw 从单对话工具，变成多 agent、多群、多场景协作系统。评论区最有价值的信息是，隔离不一定靠多个 bot，而可以靠同一个 bot + 不同群 + agent 指定，这样更不容易混乱。
